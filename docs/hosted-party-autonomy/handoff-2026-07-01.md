# Handoff: Hosted Party Autonomy, 2026-07-01

## Active Goal

Complete and verify hosted party autonomy while preserving LAN/IP access:
hosted play should invite visible nearby players to a full party, keep questing
as a coordinated group, use party chat intent to influence behavior, accept and
complete practical pre-20 quests, and eventually pass one continuous program-
clean live run to level 20.

## Slice Completed In This Handoff

This slice addressed hosted follow behavior noise and live harness classification:

- Hosted follow no longer sends `/follow` while the runtime is also driving a
  travel goal toward the leader. This avoids the live server start-stop noise
  where movement input immediately cancels follow and produces repeated
  "Now following" plus "You stop following" messages.
- The live harness now has an explicit check for hosted follow start-stop noise.
- In-game deaths, wipes, and failed pulls are now treated as gameplay events,
  not program failures. The harness still records de-duplicated player deaths
  for strategy review, but the runtime-clean gate now focuses on hosted errors
  and WebSocket errors. Stuck resets, status poll failures, party loss, and
  behavior-noise gates still matter.
- Requirements, design, state, progress, and this handoff were updated to
  reflect these rules and the post-restart validation.
